Usage

Usage​

Once you have your HelioDB set up running, you can start with creating collections, indexes and perform queries on them.

Create a collection​

HelioDB provides helio_api.create_collection function to create a new collection within a specified database, enabling you to manage and organize your BSON documents effectively.

SELECT helio_api.create_collection('Heliodatabase','patient');

Parameters​

Description
HeliodatabaseThe name of the database where the collection will be created.
first_collectionThe name of the collection to be created.

Perform CRUD operations​

Insert documents​

The insertOne() command is used to add a single document into a collection.

select 1 from helio_api.insert_one('Heliodatabase','patient', '{ "patient_id": "P001", "name": "Alice Smith", "age": 30, "phone_number": "555-0123", "registration_year": "2022","conditions": ["Diabetes", "Hypertension"]}');
select 1 from helio_api.insert_one('Heliodatabase','patient', '{ "patient_id": "P002", "name": "Bob Johnson", "age": 45, "phone_number": "555-0456", "registration_year": "2022", "conditions": ["Asthma"]}');
select 1 from helio_api.insert_one('Heliodatabase','patient', '{ "patient_id": "P003", "name": "Charlie Brown", "age": 29, "phone_number": "555-0789", "registration_year": "2023", "conditions": ["Allergy", "Anemia"]}');
select 1 from helio_api.insert_one('Heliodatabase','patient', '{ "patient_id": "P004", "name": "Diana Prince", "age": 40, "phone_number": "555-0987", "registration_year": "2023", "conditions": ["Migraine"]}');
select 1 from helio_api.insert_one('Heliodatabase','patient', '{ "patient_id": "P005", "name": "Edward Norton", "age": 55, "phone_number": "555-1111", "registration_year": "2024", "conditions": ["Hypertension", "Heart Disease"]}');
Parameters​
Description
HeliodatabaseThe name of the database where the collection exists.
patientThe name of the collection where data is inserted.
''Provide the valid json document to be inserted.

Read document from a collection​

The helio_api.collection function is used for retrieving the documents in a collection.

SELECT document FROM helio_api.collection('Heliodatabase','patient');

Alternatively, we can apply filter for specific condition using @@ or @=

SET search_path TO helio_api, helio_api_catalog,helio_core;
SET helio_core.bsonUseEJson TO true;

SELECT document FROM helio_api.collection('Heliodatabase','patient') WHERE document @@ '{"patient_id":"P005"}';

Similarly, we can also query for range using @> and @< operators.

SELECT document FROM helio_api.collection('Heliodatabase','patient') WHERE document @> '{"age":10}' and document @< '{"age":35}';

Update document in a collection​

HelioDB uses the helio_api.update function to modify existing documents within a collection.

The SQL command updates the age for patient P004.

select helio_api.update('Heliodatabase', '{"update":"patient", "updates":[{"q":{"patient_id":"P004"},"u":{"$set":{"age":14}}}]}');

Similarly, we can update multiple documents using multi property.

SELECT helio_api.update('Heliodatabase', '{"update":"patient", "updates":[{"q":{},"u":{"$set":{"age":50}},"multi":true}]}');
Parameters​
Description
HeliodatabaseThe name of the database where the collection exists.
updateThe name of the collection which contains the document to be updated.
updatesThe operation to be performed on collection.
q: {}The query to qualify records for operation. An empty object matches all documents in the collection.
uThis defines the update operation to perform.
multiIndicates that the update is applied to single or multiple matching documents.

Delete document from the collection​

HelioDB uses the helio_api.delete function for precise document removal based on specified criteria.

The SQL command deletes the document for patient P002.

SELECT helio_api.delete('Heliodatabase', '{"delete": "patient", "deletes": [{"q": {"patient_id": "P002"}, "limit": 1}]}');
Parameters​
Description
HeliodatabaseThe name of the database where the collection exists.
deleteThe name of the collection to perform the deletion from.
deletesThe operation to be performed on collection.
q: {}The query to qualify records for operation. An empty object matches all documents in the collection.
limitEnables control towards the removal of matching documents. It could either be 0 for removing all documents Or 1 for removing single record

Perform aggregations Group by​

HelioDb provides the helio_api_catalog.bson_aggregation_pipeline function, for performing aggregations over the document store.

The example projects an aggregation on number of patients registered over the years.

SELECT document FROM helio_api_catalog.bson_aggregation_pipeline('Heliodatabase', '{ "aggregate": "patient", "pipeline": [ { "$group": { "_id": "$registration_year", "count_patients": { "$count": {} } } } ] }');

We can perform more complex operations like $bucket, $addtoSet, $facet, geoNear, $graphLookup, inverseMatch, mergeObjects using the pipeline. Adding a couple of more usage examples. The example demonstrates an aggregation on patients, categorizing them into buckets defined by registration_year boundaries.

SELECT document FROM bson_aggregation_pipeline('Heliodatabase', '{ "aggregate": "patient", "pipeline": [ { "$bucket": { "groupBy": "$registration_year", "boundaries": ["2021","2023","2025"], "default": "unknown" } } ] }');

This query performs an aggregation on the patient collection to group documents by registration_year. It collects unique patient conditions for each registration year using the $addToSet operator.

SELECT document FROM helio_api_catalog.bson_aggregation_pipeline('Heliodatabase', '{ "aggregate": "patient", "pipeline": [ { "$group": { "_id": "$registration_year", "conditions": { "$addToSet": { "conditions" : "$conditions" } } } } ] }');

Join data from multiple collections​

Let's create an additional collection named appointment to demonstrate how a join operation can be performed.

select 1 from helio_api.insert_one('Heliodatabase','appointment', '{"appointment_id": "A001", "patient_id": "P001", "doctor_name": "Dr. Milind", "appointment_date": "2023-01-20", "reason": "Routine checkup" }');
select 1 from helio_api.insert_one('Heliodatabase','appointment', '{"appointment_id": "A002", "patient_id": "P001", "doctor_name": "Dr. Moore", "appointment_date": "2023-02-10", "reason": "Follow-up"}');
select 1 from helio_api.insert_one('Heliodatabase','appointment', '{"appointment_id": "A004", "patient_id": "P003", "doctor_name": "Dr. Smith", "appointment_date": "2023-03-12", "reason": "Allergy consultation"}');
select 1 from helio_api.insert_one('Heliodatabase','appointment', '{"appointment_id": "A005", "patient_id": "P004", "doctor_name": "Dr. Moore", "appointment_date": "2023-04-15", "reason": "Migraine treatment"}');
select 1 from helio_api.insert_one('Heliodatabase','appointment', '{"appointment_id": "A007","patient_id": "P001", "doctor_name": "Dr. Milind", "appointment_date": "2023-06-05", "reason": "Blood test"}');
select 1 from helio_api.insert_one('Heliodatabase','appointment', '{ "appointment_id": "A009", "patient_id": "P003", "doctor_name": "Dr. Smith","appointment_date": "2023-06-25", "reason": "Follow-up visit"}');

The example presents each patient along with the doctors visited.

SELECT document FROM helio_api_catalog.bson_aggregation_pipeline('Heliodatabase', '{ "aggregate": "patient", "pipeline": [ { "$lookup": { "from": "appointment","localField": "patient_id", "foreignField": "patient_id", "as": "appointment" } },{"$unwind":"$appointment"},{"$project":{"_id":0,"name":1,"appointment.doctor_name":1,"appointment.appointment_date":1}} ]}');

Collection management​

We can review for the available collections and databases by querying helio_api_catalog.collections.

SELECT * FROM helio_api_catalog.collections;

helio_api_catalog.collection_indexes allows reviewing for the existing indexes on a collection. We can find collection_id from helio_api_catalog.collections.

SELECT * FROM helio_api_catalog.collection_indexes WHERE collection_id = 2;

ttl indexes by default gets scheduled through the pg_cron scheduler, which could be reviewed by querying the cron.job table.

select * from cron.job;

Indexing​

Create an Index​

HelioDB uses the helio_api.create_indexes_background function, which allows background index creation without disrupting database operations.

The SQL command demonstrates how to create a single index on fields age on the patient collection of the Heliodatabase.

SELECT * FROM helio_api.create_indexes_background('Heliodatabase', '{ "createIndexes": "patient", "indexes": [{ "key": {"age": 1},"name": "idx_age"}]}');

The SQL command demonstrates how to create a compound index on fields age on the patient collection of the Heliodatabase.

SELECT * FROM helio_api.create_indexes_background('Heliodatabase', '{ "createIndexes": "patient", "indexes": [{ "key": {"registration_year": 1, "age": 1},"name": "idx_regyr_age"}]}');

The SQL command demonstrates how to create a ttl index on fields age on the patient collection of the Heliodatabase.

CALL helio_api.create_indexes('db', '{"createIndexes": "appointment", "indexes": [{"appointment_date": {"ttl": 1}, "name": "appt_ttl_idx", "v" : 1, "expireAfterSeconds": 50}]}');

Drop an Index​

HelioDB uses the helio_api.drop_indexes function, which allows you to remove an existing index from a collection. The SQL command demonstrates how to drop the index named id_ab_1 from the first_collection collection of the Heliodatabase.

CALL helio_api.drop_indexes('Heliodatabase', '{"dropIndexes": "patient", "index":"idx_age"}');
